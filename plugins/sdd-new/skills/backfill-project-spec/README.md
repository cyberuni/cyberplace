# backfill-project-spec

Non-user-invocable SDD skill holding the **project-level layout bootstrap**: when an existing project has **no consolidated spec**, choose how the spec is organized, scaffold that skeleton, and declare the choice.

Loaded via the harness (`Skill`) by the **conductor** (the main session) **once at bootstrap** when `start-mission` explore finds an existing project lacking a spec. It is **not** a user entry — the single entry is `start-mission`; this is an internal explore-phase step. It runs the six-step procedure — detect the project shape → choose the spec location (colocated / hoisted / monorepo-member) → recommend + choose a strategy (capability-first default, mirror-source) → scaffold the shared envelope + the strategy skeleton (each node declaring a legal `spec-type`) → declare the `project-path` frontmatter + the body placement map → hand back to per-unit explore — and leaves the tree at `status: draft`.

Enacts the structure-doc trio: `.agents/specs/sdd/design/spec-structure.md` (the node taxonomy — three spec-types, the concept axis, the two-level depth cap, screaming-architecture default), `.agents/specs/sdd/design/spec-layout.md` (the strategy menu, the shared envelope, the per-strategy SDD-fit, and the body-declared organization), and `.agents/specs/sdd/design/project-unit.md` (spec location, the hoist-out-of-the-distributable rule). Bakes in the two-level depth cap (sub-groupings are `concept:` tags, never a third folder level), the boundary-aligned mirror-depth rule, the spec-type classifier, the reserved by-concept index block (filled by `corpus/concept-index`, not by hand), ADR-as-facet (a `design/decisions/` log, never an ADR-organized body), and the output boundary — it authors no node's `## Use Cases`/`.feature`, renders no gate verdict, and writes no control frontmatter (those are `validate-spec`'s and the conductor's). Proposes node placement for the formation Warden to confirm.
