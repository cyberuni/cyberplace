# acceptance/ — e2e behavior suite

Cross-capability outcomes (the e2e suite, authored in explore): the **spec → write → verify** loop — a
documentation artifact goes from a `spec-writer` contract, through a `doc-writer` implementation, to a
`quill-judge` pass over every frozen scenario — plus the gate that blocks a document with any failing check.

> **This README is a `descriptive` index** — no `spec-type` marker; each e2e outcome lives in its own
> `.feature` authored in explore.
