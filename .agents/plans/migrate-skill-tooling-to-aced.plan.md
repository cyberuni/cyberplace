# migrate-skill-tooling-to-aced — CLI skill-tooling into aced as capability nodes

Internal CR (no external source) against `.agents/specs/aced/` (status: implemented). Moves the
skill-authoring/validation engines out of the cyberplace CLI into aced as self-contained node
`.mts` engines inside skill folders (SDD convention). cyberplace keeps only marketplace + adjacent
(`awesome`/`registry`/`tavern`, plus untouched `commit`/`governance`/`hook`); its `audit` + `skill`
commands are un-spec'd legacy, so their removal is clean deletion, not a spec narrowing.

## Placement (capability-first — screaming architecture, user-approved)

- `audit/validate.ts` (S/Q/E mechanical checks) -> **config-authoring/improve-skill** — a NEW node
  that finally specs the orphan `improve-skill` skill AND lands its engine. `define-skill` calls it
  as a pre-commit gate.
- `skill/source.ts` -> **contribute/contribute-skill** engine (already its caller).
- `skill/list.ts` (inline `getPackageRoot`) + `skill/repair.ts` (validate-private + repair-private,
  writes files) -> **manage/** engines (mirrors `manage-model-runners`).
- `skill/manifest.ts` `readSkillManifest` -> inline into `validate.mts` (small, node-only already).

## Convention

Self-contained `.mts`: `node:*` only, `#!/usr/bin/env node`, pure fns exported for `node:test`,
`process.stdout.write` + exit codes (no cyberplace `output.ts`), node-≥23.6 no-deps. These are
deterministic engines -> **SDD default builder + impl-judge** (node:test), not aced rubric judges.
